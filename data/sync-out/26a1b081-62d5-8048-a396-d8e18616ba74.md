## What is “Mapping on your own”?

<div class="notion-spacer" aria-hidden="true" role="presentation"></div>

When CoMapeo is first installed on a new device it is set up to start collecting data within a project of which it is the only member, it is not collaborating with any other devices or teams. This might be ideal for individuals working independently, with no need for customized categories. 

:::note 👉🏽 Note
Changing category sets requires  ![app-icon-comapeo-coordinator](assets/77ddbc4137798bed6b6b838cdfc22c990548dd8dc643056e500ffb59b0eb2b1b.png)Coordinator Tools which become visible after a project has been set up. There is no requirement for inviting devices for a project to have Customized Category Sets.
Go to 🔗 [Changing Categories Set](/docs/changing-categories-set)** **to follow necessary steps
:::

In some cases, people may want to backup and view the information they collected in a computer. To do this, install CoMapeo Desktop on the computer and invite it to the same project. This workflow works the same way as inviting a different mobile device to a project - both require the original device to invite them to collaborate. 

## Collecting Data with a Team and a Project

One of CoMapeo's strengths is to allow users to not only collect data on their own, but to collaborate with others as part of a team. To do this, users set up new projects, of which they become Coordinators, and then can invite other members as either Coordinators or Participants. These two roles have different permissions, outlined below. All users within the same project can exchange project data amongst themselves, and will have access to the same set of categories to collect data. They can also all exchange data with a Remote Archive, if it has been set up.

**What is a Team in CoMapeo**

![app-icon-comapeo-participant](assets/2dc5f8d1ce7cb4fc3056d93376d473bed776c782a82ce25d64e4cf658571a316.png) **Team** is the list of devices that are included in a project, and the roles that each device has been assigned. Teams can include anything from two devices upwards. 
Every device in the project can view the team information to know which other devices are part of their project, and every device can view all observations and tracks that have been exchanged.

## Roles available in CoMapeo

![app-icon-comapeo-participant](assets/2dc5f8d1ce7cb4fc3056d93376d473bed776c782a82ce25d64e4cf658571a316.png) **Participants** can create ![app-icon-comapeo-add-observation](assets/a29cca65239d48f05dd3abf4db7c83f36eb65cd483866c560554e9638d528329.png) ![app-icon-coordinates](assets/16ad368673fa5ccbab0930b79526b9168194d8b4af5d94145d2d0d1088a5100b.png) Observations record ![app-icon-comapeo-track](assets/bb93ade6fdc62318289db8417ed5eba91019281b6703339d0288ce4095006117.png)Tracks, and ![app-icon-edit](assets/1e513871ec012d90a9f11451bb7284263ceb07cbf1ed7d13991db190473acf23.png) **Edit** their own observations, and distribute information using ![app-icon-comapeo-exchange](assets/d1f75c78a51b8e5a813245aa74758646c26969350099762a8dc34b327ee517c7.png) **Exchange **or ![app-icon-share](assets/a419d3830b128d8df965e5057ab5ad45c01336f9ea2bead41c8ac4a2971ceef4.png) **Share **or **Download Observations.**

![app-icon-comapeo-coordinator](assets/77ddbc4137798bed6b6b838cdfc22c990548dd8dc643056e500ffb59b0eb2b1b.png) **Coordinators** can do all of the above. In addition they can  ![app-icon-edit](assets/1e513871ec012d90a9f11451bb7284263ceb07cbf1ed7d13991db190473acf23.png) Edit  observations collected by others, and  use the ![app-icon-comapeo-coordinator](assets/77ddbc4137798bed6b6b838cdfc22c990548dd8dc643056e500ffb59b0eb2b1b.png) **Coordinator tools  **including inviting collaborators, adding customized categories, and enabling features like Remote Archive.  

Roles are assigned to devices by the coordinator which invites them, during the invite process. Roles are visible on the menu, and to all project members on the team screen. 

![image](assets/60e490b502fa9ec9081b7ef3580d648b7390630f66d4f6ebc36932fda1eec6db.jpg)

:::note 👉🏽 Note
If a role has been mis-assigned, cancel the invite if possible. If the invitation has been accepted, follow steps to remove the device.  Then repeat the invite device process again being careful to select the correct role.
See 🔗 [Removing a device from a Project](/docs/removing-a-device-from-a-project)** **
:::
