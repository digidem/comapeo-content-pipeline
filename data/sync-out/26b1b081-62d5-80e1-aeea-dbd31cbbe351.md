## About Ending Projects in CoMapeo

Ending an offline project is a task with theoretical possibility due to the features included in CoMapeo, but possibly imperfect in execution depending on the team using CoMapeo, and the context they are working in. 

It is also notable that ending a project **does not remove any gathered information from any device**.  Much like share knowledge between people, CoMapeo information is not erased, but can be forgotten about.

Ending a project successfully is a matter of coordinating actions amongst the team rather than a single task. This is because CoMapeo uses a decentralized structure to store and exchange information across every device in a project. The nature of decentralized systems is that they are remarkably resilient because every device is a back-up. Ending a project is an attempt to disconnect all the backups from each other.

:::note ⚠️ Warning
In a project where there is no remote archive and teammates are not connecting to the same router, there is no way to 100% confirm that all devices have left or been removed.
:::

## Preparing to End a Project

To end a project in CoMapeo, the best practice is to archive all data and then for all team members – devices listed within the ![app-icon-comapeo-participant](assets/2dc5f8d1ce7cb4fc3056d93376d473bed776c782a82ce25d64e4cf658571a316.png) **Team** – to leave the project, with a coordinator being the last to leave. 

### Plan Data Archiving

It is good practice for a project to end with a properly archived database. 

### Clear out devices from the Team

> [!NOTE]
> Unsupported Notion block: `heading_4`

Everyone on the team but one coordinator leaves the project.

After leaving a project, if those are able to connect to the same wifi as the coordinator, the team list will be updated as a confirmation that others have left the project. 
Go to 🔗 [Leave a Project](/docs/leave-a-project) to learn more.

> [!NOTE]
> Unsupported Notion block: `heading_4`

A coordinator removes all participants and other coordinators from the project.
Go to 🔗 [Removing a device from a Project](/docs/removing-a-device-from-a-project)** **to learn more.

:::note ⚠️ Warning
The devices removed need to be connected to the same Wi-Fi router or have Remote Archive set up in order to receive the removal notification. If this does not happen, the removed devices  will continue in the project until the next time they connect with a different device in the project that has updated project information .
:::

:::note 💡 Tip
If the goal is to allow for the sole Coordinator to leave the project but also maintain the project as fully active, a different Coordinator device must be invited before the last Coordinator abandons the project.
:::

## **Abandoning a project**

The last ![app-icon-comapeo-coordinator](assets/77ddbc4137798bed6b6b838cdfc22c990548dd8dc643056e500ffb59b0eb2b1b.png) **Coordinator** to ![app-icon-comapeo-leave-project](assets/c21865fd34c324a26a9f0de7554324c29b5021c6b3bfa68b36ffac7a080dda5b.jpg) L**eave Project **is abandoning the project.  The project will no longer have a Coordinator, leaving as what can be described as a headless project. That project may have ![app-icon-comapeo-participant](assets/2dc5f8d1ce7cb4fc3056d93376d473bed776c782a82ce25d64e4cf658571a316.png) **Participants** who can still ![app-icon-comapeo-exchange](assets/d1f75c78a51b8e5a813245aa74758646c26969350099762a8dc34b327ee517c7.png) **Exchange,** but they cannot access ![app-icon-comapeo-coordinator](assets/77ddbc4137798bed6b6b838cdfc22c990548dd8dc643056e500ffb59b0eb2b1b.png) **Coordinator Tools** nor Invite nor remove other devices.

:::note 👣
### Step by Step

***Step 1: ***In the ![three-line-menu-black](assets/351bd29be717f58fba4061f4d5ef0db5508de31bd77417f31c2248b9f6cbbdee.jpg)**Menu**, select ![app-icon-comapeo-participant](assets/2dc5f8d1ce7cb4fc3056d93376d473bed776c782a82ce25d64e4cf658571a316.png) Team.

---

***Step 2:***  Select** This device**. The device in use should be the only one listed as a coordinator.

![image](assets/b00d05264dc6da4681ee9e3d4bc068f583e3e5162ceb40b458e7d120edd9221a.jpg)

---

***Step 3:*** Select ![app-icon-comapeo-leave-project](assets/c21865fd34c324a26a9f0de7554324c29b5021c6b3bfa68b36ffac7a080dda5b.jpg) **Leave Project.**

![image](assets/0e2f4ad2c98e3ec540ee218003d6605f3346bf7958fab5301049ac792f0ec0cd.jpg)

---

***Step 4:*** Review and consider recommendations before selecting ![right-arrow-black](assets/241f861b8bb8e87777c4520b6df55e46a3877c02857f4e57925486e21a8e5734.png) **Continue**

![image](assets/48e2782a4e81f9fa119612557c0e8426a0603f66e38e63a0e0dfc00d9d43af85.jpg)

---

***Step 5:*** If prepared to abandon the project, select ![app-icon-comapeo-leave-project](assets/c21865fd34c324a26a9f0de7554324c29b5021c6b3bfa68b36ffac7a080dda5b.jpg) **Yes, Leave**

![image](assets/71007df5f343bca8283d68adbd651dd896dc2a4b0c3e376143bbd9ba43ca14ab.jpg)

---

***Step 6:*** This device has left the project. Start **Mapping on your own **by opening the project with the ![app-icon-comapeo-device-name](assets/d1f67acffc237e7d29830756e6b11cfcdab7f0188883efba3a64946062eb0a4c.png)**Device Name**.

![image](assets/8628d22979b19f0dff3bc1c90638c0f434375d1406bad331b0f4ec5a5c3ab667.jpg)

---

***Step 7:*** To continue **Mapping on your own **create at least one observation. Otherwise Select ![app-icon-comapeo-switch-project](assets/ce83f9d3ea687047295a17cb3e9e090b3f7b1e1196ac2c200404927cae1c1a25.jpg) **Switch project.**

![image](assets/2b6157f64b92ca96031cf1d5bd855d28c67a9d9468fc1fd074067a6a50d8d561.jpg)

:::note 👉🏽 More
When changing project without adding an observation to the Mapping on your own project named with the device name will not save and will not appear on the list of projects.
:::

---

***Step 8:*** Open the desired project

![image](assets/0631b02b5151fac5d0e93e3404b0bac1bb2aeece901fda744f1a5bd4b1b2a4d0.jpg)
:::
